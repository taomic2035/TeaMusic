# 设计：汽水音乐式沉浸歌词播放器
- 日期：2026-06-11
- 状态：用户已确认 A+B+C 组合方向，待转实现计划
- 关联：替代 `2026-06-10-minimal-cover-immersive-ui-design.md` 中“无歌词、无沉浸播放页”的方向

## 背景与目标

TeaMusic 当前已经是 Electron + React 的 420x820 竖屏桌面播放器，并且具备本地曲库、在线找歌、封面读取、LRC 读取、媒体键和状态持久化能力。上一版方向把 UI 压缩成极简统一列表，但新的产品目标改为：参考手机端汽水音乐，把主体验重新聚焦到“正在播放”的沉浸页，并让封面、歌词和色彩成为界面核心。

目标不是复刻汽水音乐品牌或资源，而是迁移其交互气质：
- 打开就是正在播放，而不是先看到工具化列表。
- 当前歌曲/专辑封面决定整窗主色、背景光场和控件高亮。
- 歌词可见、可滚动、可逐字高亮，并有光晕扫过当前句。
- 列表、搜索、本地导入、在线找歌仍完整可用，但退到抽屉或菜单中。
- 保持桌面小窗效率：键盘、媒体键、音量、进度、喜欢、移除本地文件等能力不丢。

## 已确认设计方向

采用视觉方案页中的组合方向：

1. **A：汽水主播放页回归**作为主屏骨架。
   主屏显示顶部轻导航、大封面、歌曲信息、歌词预览、播放进度和主控件。曲库列表不再占据第一屏。

2. **B：歌词剧场**作为歌词体验。
   当前歌词行居中突出，前后歌词上下淡出；当前行支持逐字高亮和一条柔和光晕从左到右扫过。点击歌词区域进入全屏歌词模式。

3. **C：列表沉浸混合**只保留效率。
   统一列表作为上滑面板或抽屉，支持快速选歌、搜索结果、状态标记和本地/已补全 badge。列表不是默认主视觉。

## 信息架构

### 主播放屏

主屏由四层组成：

- **色彩层**：当前 `currentTrack.coverUrl` 驱动 `.app-backdrop`，并额外派生 CSS 变量控制按钮、歌词高亮、进度条和玻璃面板边缘色。无封面时回退到稳定的深色渐变。
- **封面层**：大封面位于中上部，是当前歌曲的视觉锚点。封面切换需要平滑淡入，不做夸张旋转。
- **歌词层**：歌词在封面下方或半覆盖封面底部，当前行大字号，前后行低透明度。歌词可点击。
- **控制层**：底部保留进度条、播放模式、上一首、播放/暂停、下一首、喜欢和更多菜单。

### 列表面板

列表变成“曲库抽屉”：

- 顶部搜索按钮或上滑手势打开。
- 展示 `filterTracks(tracks, query)` 的统一结果。
- 行内保留播放指示、歌名、歌手/专辑、badge。
- 点击选歌，双击或点击播放入口立即播放。
- 列表面板可关闭，关闭后返回主播放屏。

桌面没有真实手机手势时，上滑由按钮和滚轮/拖拽区兼容实现。第一版可先做按钮打开的底部抽屉，后续再加拖拽阻尼。

### 全屏歌词模式

点击主屏歌词进入歌词模式：

- 封面缩小为顶部色彩源或轻量缩略图。
- 歌词占据主体空间，当前行保持屏幕中线附近。
- 当前字按播放时间推进高亮；如果只有普通 LRC 行级时间，没有逐字时间，则用当前行在下一行开始前的时间区间均匀分配到每个字符。
- 当前行有柔和光晕扫过，扫光进度与逐字进度同步。
- Esc 或顶部返回按钮退出。

## 操控逻辑

### 播放

保留现有能力：
- 空格播放/暂停。
- 左右方向键或媒体键切歌。
- 上下方向键调音量。
- 进度条 seek。
- 播放结束按顺序、单曲循环、随机模式处理。
- 系统 Media Session 元数据和控制不变。

新增/恢复：
- 点击歌词区域进入全屏歌词。
- 在全屏歌词中点击任意歌词行跳转到该行时间。
- 列表抽屉打开时，Esc 优先关闭抽屉；歌词全屏打开时，Esc 优先退出歌词全屏；菜单/搜索/在线找歌沿用现有关闭优先级。

### 菜单

顶部菜单用于全局入口：
- 搜索曲库。
- 打开曲库列表。
- 导入本地音乐。
- 在线找歌。

底部更多菜单用于当前曲目：
- 音量。
- 移除本地音乐（仅本地曲目）。
- 在文件管理器中显示（仅可定位本地曲目）。
- 后续可扩展睡眠定时，但本轮不强制恢复。

## 色彩管理

第一版使用两级策略，避免引入过重依赖：

1. **封面模糊背景**：已有 `.app-backdrop` 继续使用当前封面作为全屏模糊背景。
2. **前景变量派生**：新增轻量取色函数，优先用 `HTMLImageElement + canvas` 从封面中心/缩略采样提取主色、亮色和深色，写入 CSS 变量：
   - `--theme-accent`
   - `--theme-accent-soft`
   - `--theme-surface`
   - `--theme-text-strong`

如果封面是跨域不可读、data SVG、file URL 读取失败或 canvas tainted，则回退到固定绿色/蓝色系，不阻塞播放。

可读性约束：
- 主文字必须保持高对比，不直接使用封面亮色作为正文色。
- 高饱和色只用于进度条、当前歌词光晕、按钮 active 态和细边框。
- 背景始终叠加暗色遮罩，避免浅色封面把歌词冲掉。

## 歌词数据流

已有 `Track.lyrics` 和 `parseLrc(content)` 保留。

新增 UI 层数据：
- `currentLyrics = currentTrack.lyrics ?? []`
- `currentLyricIndex` 根据 `playbackTime.current` 找到当前行。
- `nextLyricAt` 用下一行时间或当前音频 duration 推导。
- `lineProgress = (currentTime - currentLine.at) / (nextLyricAt - currentLine.at)`，限制在 0 到 1。
- `activeCharCount = Math.round(currentLine.text.length * lineProgress)`。

本地歌词恢复：
- 对有 `filePath` 且无 `lyrics` 的本地曲目，重新启用 `window.teaMusicBackend.readLocalLyrics`。
- 读取到 LRC 后用 `parseLrc` 写回当前 track。
- 读取失败或无 LRC 时不报错，显示占位歌词或隐藏歌词剧场的前后行，只保留当前歌曲信息。

## 动效

### 背景与封面

- 切歌时背景和封面淡入淡出，时长 450-650ms。
- 封面可有轻微缩放呼吸，但不做旋转唱片效果。
- 控件在切歌时保持位置稳定，避免布局跳动。

### 歌词滚动

- 主屏歌词显示 3-5 行，当前行自动滚到中部。
- 全屏歌词显示更多行，当前行始终靠近视觉中心。
- 使用 `transform` 和 `opacity` 做滚动过渡，避免频繁重排。

### 逐字与光晕

- 当前行内每个字符包装为 span，根据 `activeCharCount` 切换 active。
- 当前行伪元素或内部 `lyric-shine` 使用 CSS 变量 `--lyric-progress` 控制扫光位置。
- 如果用户开启系统减少动态效果（`prefers-reduced-motion: reduce`），关闭扫光和呼吸，只保留高亮状态。

## 组件边界

当前 `src/App.tsx` 已偏大。实现时允许在 `src/` 下新增小组件和 hooks，避免继续膨胀。

建议拆分：
- `src/components/ImmersivePlayer.tsx`：主播放屏、封面、底部主控件布局。
- `src/components/LyricStage.tsx`：主屏歌词和全屏歌词共用渲染。
- `src/components/LibraryDrawer.tsx`：曲库抽屉和列表。
- `src/hooks/useDominantTheme.ts`：从封面提取主题 CSS 变量。
- `src/domain/lyrics.ts`：歌词行状态、逐字进度、当前行查找等纯函数。

如果计划阶段判断拆分代价过高，可先把纯函数落在 `domain/lyrics.ts`，组件仍留在 `App.tsx`，但歌词计算和取色逻辑不能散落在 JSX 中。

## 测试策略

单元测试：
- `getCurrentLyricIndex` 在时间边界正确。
- `getLyricLineProgress` 限制在 0-1。
- `splitLyricCharacters` 对中文逐字切分稳定。
- 无下一行时使用 duration 作为结束时间。

组件测试：
- 默认渲染沉浸播放屏而非纯列表。
- 初始种子歌词显示当前/前后歌词。
- 播放进度变化后当前歌词行变化。
- 点击歌词进入全屏歌词。
- 在全屏歌词点击某行调用 seek 并更新进度。
- 打开曲库抽屉后能选歌并关闭。
- 本地曲目有 LRC 时调用 `readLocalLyrics` 并渲染解析后的歌词。
- 无封面或取色失败时仍渲染可读界面。

验证命令：
- `npm run typecheck`
- `npm test`
- `npm run build`
- `npm run dev` 后用浏览器或 Electron 小窗检查 420x820、360x620、500x900 三个尺寸。
